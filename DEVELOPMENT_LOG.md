### 1. 真正的"开箱即用"：嵌入式单文件分发

#### 老版本的痛

获取临时邮箱的脚本依赖 Python。打包后，用户经常因为没装 `python-socketio` 和 `websocket-client` 导致工具报错，不符合我"绿色免安装"的初衷。

#### 新版本的解

我利用 Rust 的宏将 Python 可执行文件直接嵌入到了二进制中！

```rust
// 编译时，将外部依赖直接"塞"进 Rust 二进制文件
const PYTHON_EXE_BYTES: &[u8] = include_bytes!("../bin/tempmail_socketio_client.exe");

// 运行时解压到临时目录执行
async fn extract_embedded_exe() -> anyhow::Result<PathBuf> {
    let temp_dir = env::temp_dir();
    let exe_path = temp_dir.join("tempmail_socketio_client.exe");
    fs::write(&exe_path, PYTHON_EXE_BYTES)?;
    Ok(exe_path)
}
```

**效果**：现在你下载的只是一个极其干净的 `.exe` 单文件。运行时工具会自动在系统 Temp 目录解压调用，退出时通过 Rust 的 `Drop` 机制自动销毁不留痕迹。你不需要配置任何环境，双击即用！

### 2. 根治"账号切换"导致的 Trae 闪退

#### 老版本的痛

点击切换账号后，拉起新的 Trae IDE 有概率会直接闪退。原因是旧代码 kill 掉旧进程后只等了 500ms，文件句柄等资源根本没释放完，新旧进程发生了"撞车"。

#### 新版本的解

重写了进程生命周期管理：

- 将优雅关闭等待时间延长至 **1000ms**
- 引入轮询机制，最多等待 **5 秒** 确认 PID 完全消失
- 强行增加 **1.5s** 的资源释放缓冲期

现在切换账号，稳如老狗，再无闪退烦恼！

**关键代码**：

```rust
pub fn kill_trae() -> Result<()> {
    // 1. 优雅关闭
    let _ = command_no_window("taskkill")
        .args(["/IM", "Trae.exe"])
        .output();
    std::thread::sleep(Duration::from_millis(1000));

    // 2. 强制关闭（如果还在运行）
    if is_trae_running() {
        let _ = command_no_window("taskkill")
            .args(["/F", "/IM", "Trae.exe"])
            .output();
    }

    // 3. 轮询检查进程是否完全退出（最多等待 5 秒）
    let start = Instant::now();
    while is_trae_running() && start.elapsed() < Duration::from_secs(5) {
        std::thread::sleep(Duration::from_millis(500));
    }

    // 4. 额外等待确保资源释放
    std::thread::sleep(Duration::from_millis(1500));
    
    Ok(())
}
```

### 3. 数据安全：导出邮箱自动脱敏处理

#### 问题描述

之前有用户反馈，导出的账号数据里，邮箱全是 `6***s@guestechsystems.com` 这种带星号的无效格式。这是因为 Trae API 返回的 `non_plain_text_email` 默认是脱敏数据。

#### 解决方案

在新版中，我在 Rust 后端增加了检测机制：导出时如果检测到脱敏星号，会自动处理避免导出无效邮箱数据，保证你换电脑导入后本地数据的干净可用。

```rust
pub fn export_accounts(&self) -> Result<String> {
    let export_data: Vec<serde_json::Value> = self.store.accounts.iter().map(|acc| {
        // 检查邮箱是否脱敏（包含*号），如果是则导出为空字符串
        let email = if acc.email.contains('*') {
            String::new()
        } else {
            acc.email.clone()
        };
        
        serde_json::json!({
            "email": email,
            // ... 其他字段
        })
    }).collect();
    
    serde_json::to_string_pretty(&export_data)
}
```
